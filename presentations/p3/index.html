Hoisting

            
Two Pass Compilation in JavaScript

            
Having access to variables before they're even declared

          

          

            
Hoisting (1)

            

              Pass 1
            

            

              All declarations are located and
              identifiers are known by the
              compiler.
            

            

              Pass 2
            

            

              Execution occurs in Pass 2.
              
Because all variables are known from Pass 1, they may be accessed before they're declared..

JavaScript global scope

            

              The global scope will hold all the variables and functions that are not defined in other functions
            

            

              

            

          


          

            
JavaScript function scope

            

              The function scope begins with function's { and ends with function's }
            

            

              

            

          


          

            
JavaScript block scope

            

              Javascript does NOT have block scoping, everything gets hoisted to the top of the function, or the global area
              
 - there is one exception from this rule: the argument of a "catch" block
