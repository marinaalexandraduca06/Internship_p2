GIT - Branching(3) - Rebasing (see notes)

The Basic Rebase

Branch Rebasing


git checkout experiment
git rebase master

Branch Rebase process

  • goes to the common ancestor of the two branches (the one you’re on and the one you’re rebasing onto)
  • gets the diff introduced by each commit of the branch you’re on, save those diffs to temporary files
  • reset the current branch to the same commit as the branch you are rebasing onto
  • finally applies each change in turn

GIT - Branching - Merge Conflicts

If you changed the same part of the same file differently in the two branches you’re merging together, Git will not be able to merge them cleanly. 


git merge iss53
    Auto-merging index.html
    CONFLICT (content): Merge conflict in index.html
    Automatic merge failed; fix conflicts and then commit the result

Your file with conflict contains a section that looks something like this 


<<<<<<< HEAD:index.html

contact : email.support@github.com

=======


 please contact us at support@github.com


>>>>>>> iss53:index.html

GIT - Branching - Fixing Merge Conflicts

Edit the file(s) with conflicts


              

              please contact us at email.support@github.com

mark each as resolved 


              git add index.html

GIT - Branching - Fixing Merge Conflicts - mergetool

Simple way to use git mergetool


              git mergetool --tool meld

# create one new file in the project /home/andrei/internship/p2/

Readme for my app:

 - Instruction 3

git add file_for_merge_conflict.txt
git commit -m “added the file for demo of fixing a merge commit using git merge tool”
# create a new feature branch
git branch br_feature1
# switch to the new branch
git checkout br_feature1
# edit file file_for_merge_conflict.txt
nano file_for_merge_conflict.txt
# alter the first line to this
 - Instruction 1

git add file_for_merge_conflict.txt
git commit -m "detailed the first instruction from the readme"
git log -3
# switch to master branch
git checkout master
# edit file file_for_merge_conflict.txt
nano file_for_merge_conflict.txt
# alter the first line to this
 - Instruction 1
 -- pay attention to the following thing !
# save and exit

git add file_for_merge_conflict.txt
git commit -m "added a warning to the first instruction from the readme"
# merge branch br_feature1 into branch master, PRODUCING A CONFLICT
